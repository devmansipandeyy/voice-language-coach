# Build Plan — Voice Language Coach → Product v1

Generated by `/plan-ceo-review` on 2026-06-08 · Mode: **Selective Expansion**

This plan turns the working demo into a more trustworthy, richer product. It
sequences a hardened **foundation** first, then two accepted features
(**E6 scenarios**, **E5 pronunciation**). The longitudinal retention layer
(accounts, progress, spaced repetition, memory) is **deferred** — see
`TODOS.md` and "NOT in scope" below.

### Full-duplex barge-in state machine (F1)

```
   ┌───────────┐  learner final utterance  ┌──────────┐  first token  ┌──────────┐
   │ LISTENING │ ────────────────────────▶ │ THINKING │ ────────────▶ │ SPEAKING │
   └───────────┘                           └──────────┘               └────┬─────┘
        ▲                                                                   │ mic stays LIVE
        │  turn done / committed                                            │ (not gated)
        └──────────── BARGE-IN ◀──── real-learner partial detected ─────────┘
             cancel turn, send "clear"   (see self-trigger guard R1)
             (client flushes playback)

   Self-trigger guard (load-bearing — see Feasibility risk R1): mic stays live
   during playback, so the agent CAN hear itself. We do NOT trust browser AEC to
   suppress that. Barge-in fires only on a REAL learner partial, defined as BOTH:
     1. >= BARGEIN_MIN_WORDS recognized words in the partial (default 2), AND
     2. partial does NOT fuzzy-match the agent's currently-spoken TTS text
        (token-overlap < BARGEIN_ECHO_OVERLAP, default 0.5) — this is what
        actually rejects echo; leaked agent speech transcribes as real words
        and would pass a word-count gate alone.
   echoCancellation stays ON as best-effort, NOT relied upon. If the guard is
   unreliable on a browser, FULL_DUPLEX=false falls back to half-duplex
   (mic gated during playback, Interrupt button only).
```

### E5 data flow with shadow paths

```
 Deepgram words[] ─▶ parse confidence ─▶ aggregate per utterance ─▶ report "clarity"
        │                  │                      │                       │
        ▼                  ▼                      ▼                       ▼
```

### Config defaults (load-bearing constants — all env-overridable)

| Key | Default | Used by | Notes |
|---|---|---|---|
| `FULL_DUPLEX` | `false` | F1 | flip per-env; off = today's half-duplex |
| `BARGEIN_MIN_WORDS` | `2` | F1 | min recognized words in a partial to count as barge-in |
| `BARGEIN_ECHO_OVERLAP` | `0.5` | F1 | reject partial if token-overlap with in-flight TTS ≥ this |
| `HISTORY_MAX_TURNS` | `12` | F5 | last 12 messages (~6 exchanges) sent to the LLM |
| `STT_RECONNECT_MAX` | `3` | F2 | reconnect attempts before surfacing failure |
| `STT_RECONNECT_BASE_MS` | `500` | F2 | exponential backoff: 0.5s, 1s, 2s |
| `WS_MAX_CONCURRENT` | `50` | F3 | total concurrent sessions before 429 |
| `WS_RATE_PER_MIN_IP` | `5` | F3 | new connections per IP per minute |
| `WS_IDLE_TIMEOUT_S` | `60` | F3 | close after no audio for this long |
| `WS_MAX_SESSION_S` | `900` | F3 | hard cap; warn at 840s, grace 60s, then close |
| `PRONUNCIATION` | `false` | E5 | flag for the clarity indicator |

## Feasibility Risks

- **R1 (HIGH) — full-duplex self-trigger.** Browser `echoCancellation` is tuned for
  WebRTC far-end streams, not for TTS played locally through WebAudio/AudioWorklet,
  so the canceller often does NOT use playback as its reference and the agent's voice
  leaks into the mic. A word-count gate does not help (leaked speech is real words).
  **Mitigation:** the echo-overlap guard (reject partials that match in-flight TTS
  text) + `FULL_DUPLEX=false` half-duplex fallback. T1 ships behind the flag; if the
  T1 verify ("agent audio alone does not self-trigger") fails on target browsers,
  default stays half-duplex. This is the one item that could not land as planned.
- **R2 (MED) — confidence ≠ pronunciation.** Deepgram confidence conflates ASR
  uncertainty, rare words, accent, and SNR; a correctly-pronounced proper noun can
  score low. **Mitigation:** label "approximate," flag only sustained low-confidence
  runs (not per-word), optionally restrict to expected vocabulary. Do NOT let it
  graduate from "approximate" without the real assessment model (TODOS P3).

## Error & Rescue Registry

| Codepath | What can go wrong | Exception | Rescued? | Action | User sees |
|---|---|---|---|---|---|
| `DeepgramSTT.receive_loop` | socket drops mid-session | `ConnectionClosed` | **N→Y (F2)** | reconnect w/ backoff (max 3), then surface | "Reconnecting…" then "Mic lost, please restart" |
| `DeepgramSTT.connect` | bad key / network | `ConnectionError`/`InvalidStatus` | **N→Y (F2)** | surface, don't start session | "Speech service unavailable" |
| `llm.stream_reply` | API error / timeout | provider error | Y (broad→typed F8) | abort turn, log | "Coach error, try again" |
| `llm.generate_report` | malformed/empty JSON | `JSONDecodeError` | Y→**+retry (F7)** | retry once, then degrade | "Couldn't build full report" + partial |
| `CartesiaTTS.synthesize` | error frame / socket | `RuntimeError`/`ConnectionClosed` | Y (broad→typed F8) | fall back to browser TTS for the turn | audio still plays |
| `/ws` | flood / abuse | n/a | **N→Y (F3)** | reject over cap | 429 / close with reason |
| report render | LLM emits HTML | n/a (XSS) | **N→Y (F7)** | escape / textContent | safe text |
| E6 scenario | prompt injection | n/a | **N→Y (P1)** | delimit + length cap; hard rules outside user text | scenario applies, rules hold |

**R1b (false-negative direction):** at `BARGEIN_ECHO_OVERLAP=0.5` + `BARGEIN_MIN_WORDS=2`,
a short genuine interruption that shares ~half its tokens with the agent's current
sentence gets dropped as echo. Tune both constants together and bias toward
accepting (lower overlap threshold) once R1 echo-rejection is proven; the Interrupt
button remains the always-works escape hatch. Tested via the echo-overlap unit test (T5).
`barge-in stale audio`: already handled — the `clear` message calls
`player.clear()` (app.js:50), which stops all queued AudioBufferSources; T1 keeps
this and adds a test. Any remaining `RESCUED=N` row is a Phase-0 blocker.

## Post-build findings (2026-06-08, from live testing)
Live testing surfaced two reliability symptoms. Root causes + fixes:
- **"Stuck mid-response"** → no timeouts in the turn pipeline; a slow/wedged
  Cartesia WS (one per sentence) or stalled LLM stream hung the turn forever,
  leaving `_speaking=True` so the mic stayed gated. **Fixed:** per-token LLM
  timeout (`LLM_TOKEN_TIMEOUT_S=20`) + per-sentence TTS timeout
  (`TTS_TIMEOUT_S=20`); on timeout the turn aborts with "took too long" and the
  mic frees. (session.py, tts.py, config.py)
- **"Text but no speech"** → text streams independently of audio; a sentence
  that yields zero Cartesia chunks shows text with no sound. **Partial fix:** a
  diagnostic log when a sentence produces 0 audio chunks, so a recurrence is
  pinpointable. Likely contributor still open: the **gating gap** (DECISION
  below).
- **DECISION NEEDED — gating gap.** The client only gates the mic once audio
  starts *playing* (`setSpeaking` on first chunk), and `_on_final` *always*
  cancels the in-flight turn (session.py). So early in a turn a stray transcript
  (echo/noise) can cancel the reply mid-stream → "stuck" / "text, no speech".
  Options: (a) ignore finals for the first ~500ms of a turn (debounce); (b) in
  `_on_final`, don't cancel a turn younger than N ms; (c) gate the mic as soon
  as the turn starts server-side, not on first audio. Needs a call before it's
  fixed — each trades off interrupt responsiveness vs spurious cancels.
